Java序列化和反序列化

Java的序列化与反序列化是什么，有什么作用，如何实现？
Java序列化的是将Java对象转换成为字节序列的过程；Java反序列化是将字节序列恢复成Java对象的过程。
我们在两个普通进程之间进行相互通信，可以发图片、音频、文本、视频等等，这些都是转成二进制的形式在网络上进行传输，那么当两个java进程在网络上相互通信，我们也会以相同的方式——将其转换成字节序列来进行传输。
大概是这样：
　　Java进程A想与Java进程B进行通信，这时，A将已有的java对象进行序列化，转换成字节序列，通过网络传输给B，B收到之后将他进行反序列化，转回Java对象，通信完成！
　　序列化和反序列化的好处：1、利用序列化实现远程通信；2、实现了数据的持久化，通过序列化可以把数据永久地保存到硬盘上（通常存放在文件里）
　　看了这么多大道理，我们现在来考虑一下如何实现Java序列化和反序列化：
　　１、首先先看API

我们知道Stream是字节流的表示，Reader和Writer是字符流的表示，要转成字节，便于传输，所以果断使用如下的两个IO流

4、Transient 关键字
Transient 关键字的作用是控制变量的序列化，在变量声明前加上该关键字，可以阻止该变量被序列化到文件中，在被反序列化后，transient 变量的值被设为初始值

5、序列化ID

虚拟机是否允许反序列化，不仅取决于类路径和功能代码是否一致，一个非常重要的一点是两个类的序列化 ID 是否一致（就是 private static final long serialVersionUID)

序列化 ID 在 Eclipse 下提供了两种生成策略，一个是固定的 1L，一个是随机生成一个不重复的 long 类型数据（实际上是使用 JDK 工具生成），在这里有一个建议，如果没有特殊需求，就是用默认的 1L 就可以，这样可以确保代码一致时反序列化成功。那么随机生成的序列化 ID 有什么作用呢，有些时候，通过改变序列化 ID 可以用来限制某些用户的使用。
