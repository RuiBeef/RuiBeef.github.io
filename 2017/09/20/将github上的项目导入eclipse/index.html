将github上的项目导入eclipse

　　当我们在github上面new好repository之后，我们就要开始做项目了。
　　就我个人来说，我还是习惯将项目导入到eclipse里面，毕竟，有IDE，编程还是方便很多的，虽然，在代码没练熟之前最好不要用。
　　好了，假定你现在已经在下载好了eclipse了（自己看教程去）
　　首先，我们先安装eclipse的git插件，新版的一般都有

如果没有的话，我们可以安装一下help-Eclipse Marketplace-在Find栏中输入egit搜索

我这个版本是第二个，然后install就行了
接着，在eclipse的Window-Preferences-Team-Git 修改一下“Default repository folder”

然后我们开始建项目了！eclipse的建项目方式，大家应该都懂吧！那我假定你们建好项目了
现在我们开始上传一下项目相关的东西，项目-右键-Team-Share Project-Git（仓库类型选择Git）-点击 next，注意，是项目下，不再是导航栏的。

把除了.project之外的所有都交上去，我们知道.project是Eclipse的一个配置文件，主要作用就是表明这个文件是一个Project，其他人下载的时候导入eclipse也会出现，我们把其他的都commit掉
接着，选中项目，右键-Team-Remote-Push，这个时候出现下面的界面

我们选择custom URI，将我们项目的uri放入

设置账号密码，Next

Source ref 选择refs/heads/master，选择分支
点击Add all Branches Spec，添加所有的分支
然后在force Update打钩，我们直接强制更新，防止github原有文件修改
这时，我们就可以到我们项目中看一下，修改成功！

另：当我们在eclipse上面修改完成后，我们可以按上述步骤将修改push到github上，如果github上面有其他人修改，我们想将其放到eclipse中，我们可以使用pull
