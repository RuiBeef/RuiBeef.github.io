github使用指南

假定现在的你已经有了github账号，并有了git及配好了ssh key
　　github来建立代码仓库——new repository：

填上项目名称，这里可以选择README，用于作为你项目的描述（我没选，因为我还不知道要做什么）
　　现在，你已经创建好了一个代码仓库，可以看到如下图的显示

按套路，我们先将他放到本地，这样的好处是，我们可以在本地写好自己的代码，等有网络时再上传，大大提高我们的效率，同时，你也不会希望每次做项目时重新到github项目上下载回本地吧。
找一个盘（我找的是E盘），右键，选择git bash，这个时候我们可以看到一个git 的命令行窗口，输入我们今天的第一句魔咒（为了方便查找，我们把名字命名和仓库相同）

接着，我们初始化仓库

建立了一个readme.txt文件,里面可以做一些描述，随便写都行
接下来，有一个好魔咒

当然，也可以用

来进行提交修改，双引号中的是你修改的备注，据说蛮重要的，比较厉害的hr会直接看你提交的时候写的备注来看你是否适合成为他们team的一员，所以，我们尽可能写详细一点，不要随意提交
我们可以用加强版的偷窥魔咒git log,这样我们就可以看到提交的作者，提交的时间，已经你刚刚输入的提交信息。
git branch也是一个不错的偷窥魔咒，我们可以看到当前有哪些分支。。。我就只有一个master，这是一个默认的分支
如果你想换其他分支，可以通过git checkout -b XX在这里表示一个分支的名称，你可以自己修改

接着，我们把它提交到远程仓库,这是最后一步了！
git remote add origin https://github.com/RuiBeef/First-project.git表明本地和远程的连接
git push -u origin master我们把它提交到远程仓库的master分支上，大功告成！
