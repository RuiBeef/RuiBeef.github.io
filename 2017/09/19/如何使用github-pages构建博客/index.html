如何使用github pages构建博客

　　最近手贱，想建个blog，但是呢，一直觉得csdn和博客园的博客样式不适合自己，希望能够自己控制自己博客的样式。所以，我尝试了使用github pages+hexo来建立自己的blog。（当然，由于技术原因，一直觉得我做的很丑。。。我有空会学一下前端，把它改好看一点的）

　　基于我在这次建blog的过程中不断掉坑，而且一些兄台的教程并没有更新，所以，我想写一下我的建立blog过程。

##github账号创建并搭建博客模板

　　首先，你需要有一个github账号，作为一个程序猿，没有github会让人觉得很掉价（其实我一开始也没有，汗）

　　作为全球最大的同性交友网站，我们要学一下英语（其实我英语很差），不然难以和别人交流（yuepao），里面sign in大家应该还是看得懂的吧，然后按它网站的指导，一步一步注册啊（用户名密码邮箱什么的），还有，千万千万记得在个人的setting中配置自己的SSH keys，据说不配后面老是要重新输入密码，反正我没试过

　　好了，现在你也步入了高大上的行列了，接着，注册一个仓库先（应该是叫仓库吧，大家都这么说Repository）


　　注意喔，要你的用户名.github.io这样才能作为你的blog，这个不要问我为什么，你看一下github pages的官网，上面就这么显示的。当然，如果你够牛，你可以采用nekyll什么的，然后自己随便起个仓库名，在gh-pages分支上面写（gh-pages分支，这也是github规定的，我也不懂，汗。。。反正我用这种方法，勉强做出来一个，但是对前端的要求比较高，不适合我这种小白）
　　然后，当你创建成功之后，在项目上选择setting，拉到下面 Github pages，这里可以选择你想要的主题Choose theme，这里有十几个丑到不行的网页模板，你可以在这里选择一个（先忍住，再丑我们待会可以改）


　　好了，现在，你的blog已经初步搭建完成了。如果你可以忍受这么丑的blog，你可以在浏览器中输入http://用户名.github.io,这样你就可以看到你的blog
　　接着，把除了.git之外的所有东西都删掉，我们换好看一点的

##环境搭建

　　首先，我们先下载node.js，它的作用主要是构建静态的页面。

　　然后是git，便于我们在本地提交,当右键任意位置，显示git bash则表示配置成功，然后我们再试一下在git bash的命令窗口输入npm，如果是no command的话就是安装node.js失败，否则成功

　　接下来，我们先创建一个文件夹，如E:/MyGitHub，这个是用于存放hexo文件，现在使用召唤魔法sudo npm install -g hexo安装hexo。
　　然后初始化hexo init会丢出一大堆奇奇怪怪的文件

　　其中的theme就是主题，我们可以登录hexo的主题网站，选择自己喜欢的主题，然后按照主题里面的描述（github-readme有说）进行，一般开始就是
npm install hexo-deployer-git --save
然后修改一下_config.yml文件就ok了。
